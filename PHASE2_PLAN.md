## 📅 第 1-2 周：通信增强

### 1.1 推送通知系统 🔔

**问题**: 用户可能错过重要更新（新匹配、认领请求）

**解决方案**: 实时推送通知

#### 功能需求
- [ ] Web Push Notifications（浏览器通知）
- [ ] Email 通知（备选）
- [ ] 通知偏好设置
- [ ] 通知历史记录

#### 触发通知的事件
1. 新匹配发现（高置信度）
2. 有人认领您的物品
3. 认领被批准/拒绝
4. 收到新评论
5. 物品被标记为完成

#### 技术实现
- **前端**: Service Worker + Push API
- **后端**: 
  - Supabase Edge Functions
  - 或 OneSignal（免费层）
  - 或 Firebase Cloud Messaging
- **数据库**: 
  ```sql
  CREATE TABLE notification_preferences (
    user_id UUID,
    push_enabled BOOLEAN DEFAULT true,
    email_enabled BOOLEAN DEFAULT true,
    match_notifications BOOLEAN DEFAULT true,
    claim_notifications BOOLEAN DEFAULT true,
    comment_notifications BOOLEAN DEFAULT true
  );
  ```

#### 预计时间: 3-4 天

---

### 1.2 直接消息系统 💬

**问题**: 用户需要在平台外沟通（不安全、不方便）

**解决方案**: 内置即时消息

#### 功能需求
- [ ] 1对1 私密聊天
- [ ] 实时消息传递
- [ ] 消息历史
- [ ] 未读消息指示器
- [ ] 图片分享
- [ ] 安全地点分享

#### 用户场景
1. 认领者向失主询问更多细节
2. 失主要求额外的验证证据
3. 协调见面时间和地点
4. 确认物品详情

#### 技术实现
- **实时通信**: 
  - Supabase Realtime（推荐）
  - 或 Socket.io
- **数据库**:
  ```sql
  CREATE TABLE conversations (
    id UUID PRIMARY KEY,
    pin_id UUID REFERENCES MapPin(id),
    user1_id UUID REFERENCES auth.users(id),
    user2_id UUID REFERENCES auth.users(id),
    created_at TIMESTAMPTZ DEFAULT NOW()
  );

  CREATE TABLE messages (
    id UUID PRIMARY KEY,
    conversation_id UUID REFERENCES conversations(id),
    sender_id UUID REFERENCES auth.users(id),
    content TEXT,
    attachments TEXT[],
    read BOOLEAN DEFAULT false,
    created_at TIMESTAMPTZ DEFAULT NOW()
  );
  ```

#### 安全考虑
- 消息加密（传输中）
- 举报/屏蔽功能
- 自动过滤不当内容
- 消息审计日志

#### 预计时间: 4-5 天

  - 物品价值范围
  - 状态（开放/已解决）
- [ ] 保存的搜索
- [ ] 搜索提醒（新匹配时通知）
- [ ] 搜索历史

#### 搜索 UI
```
┌────────────────────────────────────┐
│ 🔍 搜索失物...                     │
├────────────────────────────────────┤
│ 📅 日期: 最近 7 天      [v]       │
│ 📍 距离: 5km           [v]       │
│ 💰 价值: 所有           [v]       │
│ 📂 类别: 所有           [v]       │
│ ✓ 仅显示开放报告                  │
```

#### 技术实现
- **前端**:

- **后端**:

  ```sql
  -- 全文搜索索引
  CREATE INDEX idx_mappin_search ON MapPin 
  USING GIN(to_tsvector('english', title || ' ' || description));
  ```
